Introdução

Para que possamos customizar uma instalação do Nagios, configurar permissões em usuário e grupos de usuários devemos ter conhecimento do arquivo de configuração do CGI, abaixo abordaremos as variáveis usadas no arquivo cgi.cfg para que possamos ter um entendimento da maneira que o arquivo é formado e o que podemos fazer nesse arquivo.

 

Abaixo encontra-se cada uma das variáveis que podemos utilizar:

 

main_config_file

Isso especifica o local do seu arquivo de configuração principal . Os CGIs precisam saber onde encontrar esse arquivo para obter informações sobre informações de configuração, status atual do host e serviço, etc.

 

physical_html_path

Este é o caminho no qual os arquivos HTML do Nagios são mantidos. O Nagios assume que os arquivos de documentação e imagens (usados pelos CGIs) são armazenados em subdiretórios chamados docs/ e images/, respectivamente.

 

url_html_path

É a parte do caminho da URL usada para acessar as páginas HTML do Nagios (IP/nagios).

 

use_authentication

Esta opção controla se os CGIs usarão ou não a funcionalidade de autenticação e autorização ao determinar a quais informações e comandos os usuários terão acesso.

Se você decidir não usar autenticação, remova o comando CGI para impedir que usuários não autorizados emitam comandos para o Nagios.

Mais informações sobre como configurar a autenticação e configurar a autorização para os CGIs podem ser encontradas aqui.

  • 0 = Não use a funcionalidade de autenticação;
  • 1 = Usar funcionalidade de autenticação e autorização (padrão).

 

default_user_name

A configuração dessa variável definirá um nome de usuário padrão que pode acessar os CGIs. Isso permite que pessoas dentro de um domínio seguro (ou seja, atrás de um firewall) acessem os CGIs sem necessariamente ter que se autenticar no servidor da web. Convém usar isso para evitar a autenticação básica, se você não estiver usando um servidor seguro, pois a autenticação básica transmite senhas em texto não criptografado pela Internet.

Importante: Não defina um nome de usuário padrão, a menos que você estiver executando um servidor web seguro e tenha certeza de que todo mundo que tem acesso à CGIs foi autenticado de alguma maneira! Se você definir essa variável, qualquer pessoa que não tenha se autenticado no servidor da Web herdará todos os direitos que você atribuir a esse usuário!

  • Usuário padrão (guest).

 

authorized_for_system_information

Essa opção informa uma lista de usuários que podem visualizar informações de processo/sistema, do informações estendidas do CGI . Usuários e grupos nessas listas não estão automaticamente autorizados a emitir comandos do sistema/processo. Se você quiser que os usuários ou grupos possam ser capazes de usar comandos do sistema/processo, você deve adicioná-los aos authorized_for_system_commands ou authorized_contactgroup_for_system_commands. Mais informações sobre como configurar a autenticação e configurar a autorização para os CGIs podem ser encontradas aqui .

 

authorized_contactgroup_for_system_information

Essa opção informa uma lista de grupo de contatos que podem visualizar informações de processo/sistema, do informações estendidas do CGI .

 

authorized_for_system_commands

Essa opção informa uma lista de usuários autenticados que podem emitir comandos do sistema, por meio do CGI. Usuários e grupos nessas listas não estão automaticamente autorizados a visualizar informações do sistema/processo. Se você deseja que usuários ou grupos também possam visualizar informações do sistema/processo, inclua-os nas variáveis allowed_for_system_information ou allowed_contactgroup_for_system_information.

 

authorized_contactgroup_for_system_commands

Essa opção informa uma lista de grupo de contatos que podem emitir comandos do sistema, por meio do CGI. Usuários e grupos nessas listas não estão automaticamente autorizados a visualizar informações do sistema/processo. Se você deseja que usuários ou grupos também possam visualizar informações do sistema/processo, inclua-os nas variáveis allowed_for_system_information ou allowed_contactgroup_for_system_information.

 

authorized_for_configuration_information

Essa opção informa uma lista de usuários autenticados que podem visualizar informações de configuração no CGI de configuração. Usuários e grupos nesta lista podem visualizar informações sobre todos os hosts, grupos de hosts, serviços, contatos, grupos de contatos, períodos e comandos configurados.

 

authorized_contactgroup_for_configuration_information

Essa opção informa uma lista de grupo de contatos que podem visualizar informações de configuração no CGI de configuração. Usuários e grupos nesta lista podem visualizar informações sobre todos os hosts, grupos de hosts, serviços, contatos, grupos de contatos, períodos e comandos configurados.

 

authorized_for_all_hosts

Essa opção informa uma lista de usuários autenticados que podem visualizar informações de status e configuração de hosts (todos os hosts).

 

authorized_contactgroup_for_all_hosts

Essa opção informa uma lista de grupo de contatos que podem visualizar informações de status e configuração de hosts (todos os hosts).

 

authorized_for_all_host_commands

Essa opção informa uma lista de usuários autenticados que podem emitir comandos para todos os hosts por meio do comando CGI (navegador).

 

authorized_contactgroup_for_all_host_commands

Essa opção informa uma lista de grupo de contatos que podem emitir comandos para todos os hosts por meio do comando CGI (navegador).

 

authorized_for_all_services

Essa opção informa uma lista de usuários autenticados que podem visualizar informações de status e configuração de todos os serviços.

 

authorized_contactgroup_for_all_services

Essa opção informa uma lista de grupo de contatos que podem visualizar informações de status e configuração de todos os serviços.

 

authorized_for_all_service_commands

Essa opção informa uma lista de usuários autenticados que podem emitir comandos para todos os serviços através do CGI.

 

authorized_contactgroup_for_all_service_commands

Essa opção informa uma lista de grupo de contatos que podem emitir comandos para todos os serviços através do CGI.

 

authorized_for_read_only

Essa opção informa uma lista de usuários que possuem direitos somente leitura nos CGIs. Isso bloqueará qualquer comando de serviço ou host normalmente mostrado nas páginas CGI do extinfo. Ele também impedirá que os comentários sejam mostrados para usuários ou grupos somente leitura.

 

authorized_contactgroup_for_read_only

Essa opção informa uma lista de grupo de contatos possuem direitos somente leitura nos CGIs. Isso bloqueará qualquer comando de serviço ou host normalmente mostrado nas páginas CGI do extinfo. Ele também impedirá que os comentários sejam mostrados para usuários ou grupos somente leitura.

 

lock_author_names

Essa opção permite impedir que os usuários alterem o nome do autor ao enviar comentários, confirmações e tempo de inatividade programado a partir da interface da web. Se essa opção estiver ativada, os usuários não poderão alterar o nome do autor associado à solicitação de comando.

  • 0 = Permitir que os usuários alterem os nomes dos autores ao enviar comandos;
  • 1 = Impedir que os usuários alterem os nomes dos autores ao enviar comandos (padrão).

 

statusmap_background_image

Esta opção permite especificar uma imagem a ser usada como plano de fundo no CGI do mapa de status se você usar o método de layout de coordenadas fornecido pelo usuário. A imagem de plano de fundo não está disponível em nenhum outro método de layout. Supõe-se que a imagem reside no caminho das imagens HTML (/usr/local/nagios/share/images). Esse caminho é determinado automaticamente anexando "/images" ao caminho especificado pela diretiva physical_html_path. Nota: O arquivo de imagem pode estar no formato GIF, JPEG, PNG ou GD2, no entanto, o formato GD2 (de preferência não compactado) é recomendado, pois reduzirá a carga da CPU quando o CGI gerar a imagem do mapa.

default_statuswrl_layout

refresh_rate

Esta opção permite especificar o número de segundos entre as atualizações da página para os status, statusmap e extinfo CGIs.

Padrão é 90 segundos

 

ping_syntax

Essa opção determina qual sintaxe deve ser usada ao tentar executar o ping de um host da interface WAP (usando o CGI statuswml. Você deve incluir o caminho completo para o binário de ping, além de todas as opções necessárias. A macro $HOSTADDRESS$ é substituída pela endereço do host antes que o comando seja executado.

 

escape_html_tags

Esta opção determina se as tags HTML na saída do host e do serviço (plugin) são escapadas nos CGIs. Se você habilitar essa opção, a saída do seu plug-in não poderá conter hiperlinks clicáveis (Status do serviço).

  • 0 = Desabilitado;
  • 1 = Habilitado (padrão).

 

notes_url_target

Esta opção determina o nome do destino do quadro no qual as URLs de anotações devem ser exibidas. As opções válidas incluem _blank, _self, _top, _parent ou qualquer outro nome de destino válido.

 

action_url_target

Essa opção determina o nome do destino do quadro no qual os URLs de ação devem ser exibidos. As opções válidas incluem _blank, _self, _top, _parent ou qualquer outro nome de destino válido.

 

enable_page_tour

Esta opção permite ativar/desativar o tour da página que aparece no canto inferior direito da tela. As opções válidas incluem 0 ou 1, o padrão está ativado. Versão: 4.4.0+

 

enable_splunk_integration

Esta opção determina se a funcionalidade de integração com o Splunk está ativada na interface da web. Se ativado, você verá os links "Splunk It" em vários locais nos CGIs (arquivo de log, histórico de alertas, detalhes de host/serviço, etc.). Útil se você está tentando pesquisar por que ocorreu um problema específico. Para mais informações sobre o Splunk, visite http://www.splunk.com/.

 

splunk_url

Esta opção é usada para definir o URL base para sua interface do Splunk. Essa URL é usada pelos CGIs ao criar links se a opção enable_splunk_integration estiver ativada.

 

Audio Alerts

As opções abaixo permitem especificar um arquivo de áudio que deve ser reproduzido no seu navegador se houver algum problema quando você estiver visualizando o status CGI. Se houver problemas, o arquivo de áudio para o tipo mais crítico de problema será reproduzido. O tipo mais crítico de problema está em ou mais hosts inacessíveis, enquanto o menos crítico é um ou mais serviços em um estado desconhecido.

Presume-se que os arquivos de áudio estejam em media/ subdiretório do diretório HTML (/usr/local/nagios/share/media).

 

Exemplo:

  • host_unreachable_sound=hostu.wav

    Som que será executado quando um host não puder ser alcansado.

     

  • host_down_sound=hostd.wav

    Som que será executado quando um host estiver down.

     

  • service_critical_sound=critical.wav

    Som que será executado quando um serviço estiver crítico.

     

  • service_warning_sound=warning.wav

    Som que será executado quando um serviço estiver Warning.

     

  • service_unknown_sound=unknown.wav

    Som que será executado quando um serviço for desconhecido (apenas o resultado do check).

 

Statusmap CGI Color Transparency Indexes

Essas opções definem os valores r, g, b da cor de plano de fundo e usam o CGI do mapa de status; portanto, navegadores normais que não podem mostrar transparência png real definem a cor desejada como cor de plano de fundo (para torná-la bonita). O padrão é branco: (R, G, B)=(255.255.255).
